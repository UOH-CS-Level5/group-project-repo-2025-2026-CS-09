This project uses Agile Scrum practices with documented user stories, epics, ceremonies, and a GitHub Project Board for full transparency.

## SCRUM Process

### Stand-Ups (Weekly)
We held stand-ups every few days to weekly, suitable for a small, part-time team.

### Sprint Planning
- Defined sprint goals
- Broke down epics into actionable stories

### Sprint Review
- Demoed completed increments  
- Collected feedback from mock stakeholders  
- Adjusted backlog accordingly  

### Retrospective
We reflected on:
- What went well  
- What didn’t  
- Improvements for next sprint
